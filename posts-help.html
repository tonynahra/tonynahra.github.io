Guide: How to Create Post Cards

A reference guide for the custom parameters in posts.html.

1. The Card `div` Structure (The Template)

This is your main template. Copy and paste this structure into posts.html to create a new card.

<!-- Card Item -->
<div class="card-item" data-category="CategoryName" style="border-left-color: #ColorHex;">
    
    <!-- Optional: Preview Image -->
    <img src="path/to/image.png" class="card-image" alt="Preview">
    
    <!-- Title (Used for search) -->
    <h3>My Post Title</h3>
    
    <!-- Description (Used for search) -->
    <p>A short description of the post or link.</p>
    
    <!-- Visual Category Label -->
    <span class="card-category">CategoryName</span>
    
    <!-- The Link: See Section 3 for options -->
    <a href="path/to/content.html" data-load-type="html">Read More →</a>
    
</div>

2. Card `div` Parameters

These parameters control the filtering and appearance of the card itself.

  • data-category="CategoryName"
    This is the *most important* attribute for filtering. The JavaScript `filterPostCards()` function reads this value. It must match one of the <option value="CategoryName"> tags in your filter dropdown.
  • style="border-left-color: #ColorHex;"
    This controls the color of the accent border on the left.

3. The Link `<a>` Tag (The "Command")

This is the most powerful part. The attributes on the <a> tag control what happens when the card is clicked.

A) Standard Link (Opens in New Tab)

If you just want the card to open a link in a new browser tab, **do not** add any data-load-type attribute. (Requires the JavaScript update below).

<!-- This link will open in a new tab -->
<a href="https://www.google.com">View on Google →</a>

B) Dynamic Link (Loads in Right-Hand Area)

To load content into the right-hand #content-area, add the data-load-type attribute.

  • data-load-type="html"
    Loads a local HTML file (like a full blog post you wrote). The file must be an HTML *fragment* (no <html> or <body> tags).
  • data-load-type="image"
    Loads a single image (local or remote) into the content area, centered and safely scaled.
  • data-load-type="iframe"
    Loads an entire remote website (like Wikipedia). This is the best solution for showing remote pages, as all their relative links and images will work.
  • data-height="..." (Optional)
    Only works with data-load-type="iframe". Overrides the default height for a specific iframe (e.g., data-height="1200px").

4. The Filter Bar (in `posts.html`)

To add a new category to your filter, you must add a new <option> tag to the <select id="post-category-filter"> dropdown. The value must exactly match the data-category on your cards.

<select id="post-category-filter">
    <option value="all">All Categories</option>
    
    <!-- Add your new category here -->
    <option value="Python">Python</option>
    <option value="Data">Data</option>
</select>

5. **Required JavaScript Update**

To make the new, simpler click logic work, you **must replace** the .card-item click handler in your common/mainPage.js file. (This code is provided in the guide.md file).
